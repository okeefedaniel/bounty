Bounty User Manual

Comprehensive guide to federal-funds discovery, matching, and tracking.

FAQ

Overview

Bounty pulls federal grant opportunities from Grants.gov, scores them against your agency's priorities with AI, and gives a coordinator the tools to track each opportunity through preparation, internal approval, application, and award. Awarded opportunities hand off to Harbor for program management.

Public catalog

Federal opportunities synced from Grants.gov, browsable without login.

AI matching

Claude-scored relevance against personal + state preferences.

Coordinator pipeline

Watching → Preparing → Approved → Applied → Awarded, with collaborators and attachments.

Suite handoffs

Manifest signs internal approval; Harbor receives the awarded program.

Roles

Bounty roles are issued via Keel SSO and surfaced in your sidebar profile.

RoleCapability
Admin Full access. Manage state preferences, all tracked opportunities, Harbor connection, demo data.
Coordinator Drive opportunities. Claim, invite, transition, send for signing, push to Harbor, manage state preferences.
Analyst Track, run AI matching, comment, attach files on opportunities they're invited to.
Viewer Read-only across the dashboard and opportunities they're invited to.

Suite-level system_admin inherits Admin behavior on Bounty.

Getting Started

Signing in
1Visit https://bounty.docklabs.ai/ (or your tenant's host).
2Click Sign in with DockLabs (the suite OIDC button).
3You'll land on /dashboard/.
If you're already signed in to another DockLabs product, the redirect is seamless — no second login form.
What you'll see first
  • Dashboard — summary of your tracked opportunities, new high-relevance matches, and pending actions.
  • Sidebar — Federal Opportunities (public catalog), Tracked, AI Matching (if you have AI access), and Harbor Connection (coordinators).

Federal Opportunities (Public Portal)

/ and /opportunities/ — the public-facing catalog of every federal opportunity Bounty has cached from Grants.gov. No login required.

Each opportunity carries:

  • Title, opportunity number (NOFO), and opportunity ID.
  • Agency name and code.
  • Category and funding instrument (grant, cooperative agreement, procurement contract, other).
  • CFDA numbers (Catalog of Federal Domestic Assistance).
  • Award range — floor, ceiling, expected number of awards, total funding pool.
  • Post date, close date, and archive date.
  • Eligibility — applicant types and free-text narrative.
  • Status — Posted, Closed, Archived, or Forecasted.
  • Grants.gov URL — deep link to the source listing.

Search & filter

  • Free-text search on title and description.
  • Filter by agency, status, and category.
  • Instant typeahead — /opportunities/instant/?q=….
  • AI chat search — /opportunities/chat/?q=… returns a Claude narrative match.

Tracked Opportunities

/tracked/ — the coordinator pipeline. Each row represents a federal opportunity your team is actively working.

Each tracked opportunity carries the source federal opportunity, the current principal driver (tracked_by), status, priority, notes, collaborators, attachments, full status history, and Harbor push status.

Lifecycle

Bounty follows the DockLabs Project Lifecycle Standard:

claim → invite collaborators → diligence (notes + attachments) → stage progression → optional handoff to Manifest for signing → signed-doc roundtrip → optional downstream export to Harbor

watching → preparing → approved → applied → awarded
                                 \         \→ declined
                                  → applied (no approval)
preparing → watching (back)
approved → preparing (back)
watching → declined (direct decline)

approved is the gated state where Manifest signing fires. Use it when your agency requires an internal sign-off on the application package before submission.

Tracking a new opportunity

1From a public opportunity detail page, click Track this opportunity.
2Bounty creates a tracked record with you as the principal driver, status Watching, and opens an assignment row recording the claim.
3The opportunity is now visible at /tracked/.

Claim & release

  • Claim — make yourself the driver. Closes any prior open assignment, opens a new one with assignment_type=CLAIMED (or MANAGER_ASSIGNED if a manager assigns on someone else's behalf).
  • Release — return the opportunity to the unowned pool. Closes the active assignment with status=RELEASED.

AI Matching

/matching/recommendations/ — AI-scored matches between your preferences and every open federal opportunity. Available to users with an Anthropic API key configured (personal on profile, or the suite-wide ANTHROPIC_API_KEY).

Each match carries:

  • Relevance score (0–100) from Claude.
  • Explanation — short narrative for fit/no-fit.
  • Status — New, Viewed, Saved, Dismissed.
  • Feedback — thumbs up/down with reason (wrong focus, budget too large/small, already aware, not eligible, not relevant, other). Feedback feeds future scoring.

Score thresholds

SettingDefaultMeaning
GRANT_MATCH_MIN_SCORE60Minimum stored as a match.
GRANT_MATCH_NOTIFY_SCORE75Fires grant_match_found.
GRANT_MATCH_HIGH_SCORE90Fires grant_match_high_score (priority high). Threshold for Helm's Awaiting Me column.

Running matching

  • Run matching — /matching/recommendations/run/ — POST scans all open federal opportunities against your preferences.
  • Mark viewed — flips your NEW matches to VIEWED.
  • Dismiss — /matching/dismiss/<id>/.
  • Track and dismiss — /matching/track-dismiss/<id>/ — track the underlying opportunity and dismiss the match in one click.
  • Feedback — /matching/feedback/<id>/ — thumbs up/down with optional reason.

Match & State Preferences

Match Preferences (personal)

/matching/preferences/ — your personal preferences for AI matching.

  • Focus areas — multi-select: Education; Health & Human Services; Environment & Energy; Infrastructure & Transportation; Public Safety; Housing & Community Development; Economic Development; Arts & Culture; Technology & Innovation; Agriculture & Food; Workforce Development; Justice & Legal Services; Other.
  • Keywords — free-text terms describing your priorities.
  • Funding range — minimum and maximum dollar amounts.
  • Description — narrative paragraph the AI uses for context.
  • Digest frequency — None, Daily, or Weekly. Cadence of match-summary emails.

State Preferences (shared)

/matching/state-preferences/ — coordinator-only. The state-wide matching baseline that every user inherits. Same fields as Match Preferences. Only one is active at a time.

When matching runs, Bounty merges your personal preferences with the active state preference. Individual users inherit the state-wide baseline without retyping it.

Collaboration

Per-opportunity roles

RoleWhat they can do
LEADDrive the opportunity. Transition status, manage collaborators, send for signing, push to Harbor.
CONTRIBUTORUpdate notes, attach files, comment.
REVIEWERRead everything; comment on notes and attachments.
OBSERVERRead-only access.

Only LEAD / CONTRIBUTOR / REVIEWER / OBSERVER are valid (suite-wide convention).

Inviting collaborators

/tracked/<id>/collaborate/ — pick a user (or enter an email for an external invitee) and a role. The invitee gets an in-app notification.

Attachments

/tracked/<id>/attachments/ — drag-and-drop file uploads. Each carries a source (manual upload, manifest_signed, etc.) and a visibility flag. Files attached as manifest_signed are produced automatically by the Manifest signing roundtrip — you never upload them by hand.

Internal Approval & Manifest Signing

When an agency requires an internal sign-off on the application package before submission, Bounty hands the package to Manifest for an ordered-signer workflow.

Send for signing

/tracked/<id>/sign/send/ — LEAD only. Available when the opportunity is in preparing status and Manifest is configured (MANIFEST_URL + MANIFEST_API_TOKEN).

When Manifest finishes the packet, the inbound webhook fires. Bounty:

  1. Files the signed PDF as an OpportunityAttachment with source=manifest_signed and the Manifest packet UUID.
  2. Transitions the opportunity to approved.
  3. Notifies the LEAD and active collaborators.
  4. Registers the attachment for FOIA export.

Local-sign fallback

/tracked/<id>/sign/local/ — when Manifest isn't configured (standalone deployment), the LEAD can upload a manually-signed PDF instead. The attachment is filed with source=local_signed and the opportunity transitions to approved. The Send-for-signing button is hidden when Manifest isn't available — there are no silent no-ops.

Harbor Handoff (Award)

When an opportunity is awarded, the LEAD can push it into Harbor as a GrantProgram so program staff can manage applications, awards, and compliance there.

Configure Harbor

/integration/harbor/settings/ — coordinator-only. Provide the Harbor URL and API token.

Push to Harbor

/integration/harbor/push/<id>/ — LEAD-only. Available when the opportunity is in awarded status and Harbor settings are configured.

The push is best-effort. On success, tracked.harbor_program_id is set to Harbor's program ID and harbor_push_status flips to Pushed. On failure, status flips to Failed with a retry control. The button is hidden when Harbor isn't configured.

Helm Inbox Integration

Helm's per-user Awaiting Me column queries Bounty's /api/v1/helm-feed/inbox/ endpoint and surfaces, for each user:

  • NEW high-relevance matches — OpportunityMatch rows with status=NEW and relevance_score >= GRANT_MATCH_HIGH_SCORE (default 90), capped at 50, ordered by score descending.
  • Unread notifications.

Each item carries a deep link back into Bounty (typically /matching/recommendations/) so the user can act with one click. The aggregate feed at /api/v1/helm-feed/ powers Helm's "Across the suite" cards.

Programmatic API

/api/v1/ — DRF resource endpoints. Auth: per-user Bearer token (SHA-256 hashed; set on BountyProfile).

ResourceEndpoint
Federal opportunities (read)/api/v1/opportunities/
Tracked opportunities/api/v1/tracked/
Match preferences/api/v1/preferences/
Opportunity matches/api/v1/matches/
Helm feed (aggregate)/api/v1/helm-feed/
Helm feed (per-user inbox)/api/v1/helm-feed/inbox/?user_sub=…

Pagination envelope: {total, limit, offset, results}. Default 50, max 200. OpenAPI schema at /api/schema/; Swagger UI at /api/docs/; ReDoc at /api/redoc/.
